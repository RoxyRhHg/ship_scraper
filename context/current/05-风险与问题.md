# 05 — 风险与问题

## 开放问题

### Q1: maritimequest/wikimedia 能否贡献足够的新图片？
- **影响**：v10 目前 0 净新增（全部与 v9.5 SHA1 重复）。修复后依赖 mq/wiki 提供差异化数据
- **状态**：待烟雾测试验证
- **风险**：如果 mq/wiki 候选也大部分重复，需要扩展新数据源

### Q2: navsource.org 永久不可达？
- **影响**：少了一个数据源
- **状态**：观察中
- **调查**：DNS 解析失败，直连和代理均不可达。可能是域名被墙或已废弃
- **缓解**：maritimequest 和 wikimedia 可部分替代

### Q3: v10 与 v9.5 候选 URL 集高度重叠
- **影响**：bing 搜索结果趋于稳定，新增 URL 有限
- **状态**：已知
- **缓解**：
  - 修复后 maritimequest/wikimedia 提供不同URL来源
  - 可考虑增加 bing 搜索词多样性
  - 可考虑使用不同的搜索引擎（google, yandex）

### Q4: 系统代理稳定性
- **影响**：代理 127.0.0.1:7890 是 Clash/V2Ray 本地代理，依赖外部节点
- **状态**：当前稳定（5/5 测试URL成功）
- **缓解**：提供 `--no-proxy` 选项用于直连源

## 已知风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| maritimequest BFS 爬取超时 | 中 | 该源贡献0候选 | 增加 mq_fetch_workers 或减少 BFS 深度 |
